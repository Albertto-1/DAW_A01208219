Preguntas:
  1. ¿Por qué es importante hacer un session_unset() y luego un session_destroy()?
    • Es una buena práctica primero limpiar todas las variables de la sesión y ya después eliminarla.
  2. ¿Cuál es la diferencia entre una variable de sesión y una cookie?
    • Una cookie se almacena en el cliente (en su navegador por un tiempo) y las variables de sesión están en el servidor durante lo que dure la sesión.
  3. ¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?
    • Le agrega la fecha al nombre para que sean diferentes y no se sobreescriba.
  4. ¿Qué es CSRF y cómo puede prevenirse?
    • Es un tipo de exploit malicioso de un sitio web en el que comandos no autorizados son transmitidos por un usuario en el cual el sitio web confía. Puede prevenirse con diferentes métodos como: Doble-posted cookie (envío doble de cookie), Unique form nonce (formulario único para cada solicitud), Requerir credenciales de autentificación nuevamente, One time Passwords. Pienso que esta última aunque es tediosa es la mejor.
