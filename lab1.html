Descripción:

URL Es la referencia/dirección de un recurso que está en la internet.
MIME Es un estandar del internet que extiende el formato de un email, permitiendole soportar caracteres distintos a los de ASCII así como soportar videos, audio, etc.
Métodos y códigos de errores HTTP Los métodos y códigos de error son los que nos ayudan a trabajar en HTML ya que facilitan el manejo de información y la retro a los usuarios.
XML Es un lenguaje para transportar información en la web. A diferencia de html, XML se encarga de transportar más que de mostrar.
XHTML Es otro tipo de lenguaje es es como la unión de HTML y XML.
HTML5 Lenguaje de programación para mostrar contenido en la web. También es la última versión de HTML
JS Es una herramienta de programación para darle comportamiento a una página web
php Es como un lenguaje-herramienta compatible con html que ayuda al desarrollo web
jsp Es una herramienta que ayuda a crear aplicaciones web que se puedan ejecutar en multiples plataformas
Ajax Técnica que permite comunicar de manera asincrona a un servidor y un navegador en formato XML.

Preguntas:

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet World Wide Web
El internet es una red de redes inmensa en donde todas las computadoras que acceden al internet están conectadas WWW es la herramienta que nos ayuda a accesar a la información que está en la internet

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Método HTTP: Propósito
GET Se usa para solicitar la representación de un recurso.
HEAD Al igual que GET, también solicira un recurso pero no su representación.
POST Se usa para enviar información. Se usa al llenar formularios.
PUT Este método puede actualizar y crear contenido.
PATCH Se utiliza para actualizar parte del contenido de un sitio, dejando al resto intacto.
DELETE Elimina un resurso.

Significa que el atributo aún es soportado pero que es mejor no usarlo ya. Un ejemplo es hgroup que se usaba para agrupar el título y el subtítulo de algo. Otros elementos depreciados son: center, command, scoped, etc.

Tiene mejoras API (getElementsByClassName() y innerHTML)

Investigación 1: Ciclo de vida y de desarrollo de los sistemas de información

Ciclo de vida de los sistemas de información

Consiste en le serie de pasos que se tienen que realizar para el desarrollo de un sistema. Estos paso son:

Planeación En esta etápa, se especifican las carácteristicas del problema a solucionar y de las posibles soluciones.
Desarrollo (Ciclo de desarrollo) Aquí se lleva a cabo la ingeniería de software (requerimientos, procesos, diagramas, implementación, pruebas, etc).
Operación Cuando el software ya está instalado y en uso
Mantenimiento En caso de fallas/actualizaciónes darle mantenimiento

Ciclo de desarrollo de sistemas de información

Literalmente es el desarrollo del proyecto hasta el punto que es entregado.

Análisis Averiguar qué es exactamente lo que tiene que hacer el sistema. La etapa de análisis en el ciclo de vida del software corresponde al proceso mediante el cual se intenta descubrir qué es lo que realmente se necesita y se llega a una comprensión adecuada de los requerimientos del sistema.
Diseño Se han de estudiar posibles alternativas de implementación para el sistema de información que hemos de construir y se ha de decidir la estructura general que tendrá el sistema (su diseño arquitectónico). El diseño de un sistema es complejo y el proceso de diseño ha de realizarse de forma iterativa.
Construcción Seleccionar las herramientas adecuadas, un entorno de desarrollo que facilite nuestro trabajo y un lenguaje de programación apropiado para el tipo de sistema que vayamos a construir. La elección de estas herramientas dependerá en gran parte de las decisiones de diseño que hayamos tomado hasta el momento y del entorno en el que nuestro sistema deberá funcionar.
Pruebas Tiene como objetivo detectar los errores que se hayan podido cometer en las etapas anteriores del proyecto (y, eventualmente, corregirlos). La búsqueda de errores que se realiza en la etapa de pruebas puede adaptar distintas formas, en función del contexto y de la fase del proyecto.
Instalación Debemos de planificar el entorno en el que el sistema debe funcionar, tanto hardware como software: equipos necesarios y su configuración física, redes de interconexión entre los equipos y de acceso a sistemas externos, sistemas operativos y bibliotecas.
